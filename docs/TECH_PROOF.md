# 기술 검증 01

## 검증 질문
React의 DOM UI와 Phaser 캔버스를 한 화면에 겹쳤을 때, 카드 드래그·타깃 앵커·전투 연출을 안정적으로 연결할 수 있는가?

## 구현 장면

### 일반 기술
`틈새 베기`를 돌짐승 몸에 놓는다.

예상 흐름:
- 카드 원본은 손패에 남음
- 물리적 궤적만 커서를 따라감
- 유효 대상에 스냅
- 주인공 전진
- 검격과 히트스톱
- 적 흔들림과 피해 숫자
- 적 몸에 틈 잔류
- 주인공 복귀

### 언령
`멎는다.`를 `내리친다` 행동에 놓는다.

예상 흐름:
- 화면 채도 감소
- 적 행동 앵커만 활성화
- 글자와 옥색 선이 대상을 향함
- 글자가 행동에 도달
- 공격 문장과 준비 자세가 멈춤
- 피해 없음 표시

## 통과 기준
- 드래그 포인터와 Phaser 대상의 좌표가 어긋나지 않는다.
- 1280×720과 작은 브라우저 창에서 모두 스냅이 된다.
- 공격 사건이 순서대로 재생된다.
- 일반 기술과 언령의 드래그가 시각적으로 다르다.
- `npm test`와 `npm run build`가 통과한다.
- 브라우저에서 30fps 이하로 눈에 띄게 끊기지 않는다.

## 실패 시
- 혼합 구조를 곧바로 폐기하지 않는다.
- 먼저 앵커 좌표를 정규화 좌표로 고정하고 드래그 연출을 단순화한다.
- 1일 안에 해결되지 않으면 React-only 시네마틱 방식으로 복귀한다.
